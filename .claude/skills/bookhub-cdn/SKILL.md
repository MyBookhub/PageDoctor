# bookhub-cdn

This skill covers the `MyBookhub/mini-cdn` repo: a CloudFormation stack
deployed via AWS SAM that provisions an S3 bucket + CloudFront distribution
serving the bookhub assets.

## Stack

- AWS SAM CLI (`sam`)
- CloudFormation (deployed as stack `bookhub-mini-cdn`)
- AWS profile: `prod`
- Region: `eu-central-1`
- Architecture: S3 bucket `bookhub-assets` (private, BlockPublicAccess) +
  CloudFront distribution with Origin Access Control (OAC) — only CloudFront
  can read the bucket

## Files

- `template.yaml` — the entire stack (Parameters, Resources, Outputs)
- `samconfig.toml` — local deploy config, gitignored
- `example.samconfig.toml` — committed template for `samconfig.toml`
- `contract.md` — manifest endpoint contract for the sync script in the
  website repo
- `README.md` — setup + deploy

## Standard workflow

```bash
sam validate --profile prod
sam deploy --profile prod        # uses samconfig.toml
aws cloudformation describe-stacks \
  --stack-name bookhub-mini-cdn \
  --profile prod \
  --query 'Stacks[0].Outputs'
```

For first deploy or to regenerate config, run `sam deploy --guided
--profile prod`.

## Manifest endpoint

`GET https://<distribution-domain>/_manifest/v1.json` — JSON listing of
assets, generated by the sync script in `MyBookhub/website`. CloudFront
serves it via a dedicated `/_manifest/*` cache behavior using the
`CachingDisabled` managed policy (TTLs all 0 — always fresh from origin).
Contract: `contract.md` at repo root.

## Cache invalidation (when needed)

Default behavior uses `CachingOptimized` (24h TTL). To force-refresh:

```bash
aws cloudfront create-invalidation \
  --distribution-id "$(aws cloudformation describe-stacks \
    --stack-name bookhub-mini-cdn --profile prod \
    --query 'Stacks[0].Outputs[?OutputKey==`DistributionId`].OutputValue' \
    --output text)" \
  --paths "/print-template/*" \
  --profile prod
```

The manifest behavior doesn't need invalidation (TTL = 0).

## Adding a new resource

1. Edit `template.yaml`, add resource under `Resources:`
2. `sam validate --profile prod`
3. `sam deploy --profile prod` — note `confirm_changeset = false`, so the
   changeset auto-applies; review the template diff before deploying
4. Verify outputs: `aws cloudformation describe-stacks ...`

## Gotchas

- **Bucket DeletionPolicy is `Delete`.** `sam delete` fails if the bucket
  has any objects/versions. Purge first:
  `aws s3 rm s3://bookhub-assets --recursive --profile prod`, then purge
  versions, then `sam delete`.
- **State lives in CloudFormation**, not on disk. The stack IS the state.
  Use `aws cloudformation detect-stack-drift` to catch drift.
- **ACM cert for custom domain must be in `us-east-1`** (CloudFront
  requirement). Phase 2 work — needs cross-region cert reference.
- **Manifest is publicly readable** — never list private/internal paths in
  it.
- **`sam deploy` first time** runs `--guided` interactively unless
  `samconfig.toml` already has `[default.deploy.parameters]`.
